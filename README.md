# Telegram RAG Bot for Book Search

Телеграм-бот для хакатонного кейса «Умный поиск по книгам». Бот умеет:

- загружать новые книги в формате `.txt`;
- хранить библиотеку локально и показывать карточки книг;
- удалять книги из библиотеки;
- искать релевантные фрагменты по запросу;
- отвечать на вопросы по книгам с цитатами-основаниями;
- отвечать только через LLM + RAG через OpenRouter.

## Стек

- Python 3.9+
- `aiogram` 3
- SQLite для каталога книг и чанков
- локальный retrieval без внешней векторной БД
- OpenRouter API для генерации ответа поверх найденного контекста

## Архитектура

Решение построено как RAG-пайплайн:

1. Книга загружается как `.txt` и сохраняется в `library/`.
2. Текст нормализуется, разбивается на чанки с overlap.
3. Метаданные книги и чанки индексируются в SQLite (`data/catalog.sqlite3`).
4. Retrieval использует гибридный скоринг:
   - BM25-подобный lexical search,
   - покрытие ключевых терминов,
   - бонус за совпадение по предложениям,
   - символьные n-граммы для устойчивости к формулировкам,
   - диверсификацию результатов через MMR-подобный отбор.
5. Для Q&A используется advanced RAG-пайплайн:
   - multi-query query transformations,
   - RAG-fusion через reciprocal-rank-style слияние кандидатов,
   - self-check / corrective retry через LLM, если первый retrieval оказался слабым.
6. Финальный ответ всегда формируется LLM по найденным цитатам.

## Структура проекта

- `bot.py` — точка входа
- `logic.py` — Telegram handlers и UI
- `storage.py` — работа с библиотекой, SQLite и чанками
- `rag_engine.py` — retrieval и answering
- `llm_client.py` — OpenRouter client
- `text_utils.py` — нормализация текста и вспомогательные функции
- `library/` — книги

## Как запустить

1. Создайте виртуальное окружение.
2. Установите зависимости:

```bash
pip install -r requirements.txt
```

3. Задайте токен Telegram-бота:

```bash
export BOT_TOKEN="ваш_telegram_token"
```

4. Запустите бота:

```bash
python3 bot.py
```

Фанфакт: Telegram username этого бота — `@jatbro_bot`.

## Как включить LLM через OpenRouter

OpenRouter обязателен. Без него бот не стартует.

```bash
export OPENROUTER_API_KEY="ваш_openrouter_api_key"
export OPENROUTER_MODEL="укажите_модель_опционально"
```

После этого перезапустите бота.

Что нужно сделать вам:

1. Зарегистрироваться в OpenRouter.
2. Получить API key.
3. Опционально выбрать модель и прописать ее в `OPENROUTER_MODEL`.
4. Перезапустить `python3 bot.py`.

Если задан только `OPENROUTER_API_KEY`, бот по умолчанию использует `openrouter/free`.
Если конфигурация OpenRouter невалидна, бот не запустится.

## Как загружать книги

- Нажмите в боте `Добавить книгу`.
- Отправьте `.txt`-файл документом.
- Бот сохранит книгу в `library/` и автоматически перестроит индекс.

Также можно заранее положить `.txt`-файлы в папку `library/` и при старте бот их проиндексирует.

## Что видно в библиотеке

Для каждой книги бот показывает:

- название;
- автора;
- дату добавления;
- дату последнего обновления индекса;
- размер файла;
- количество слов;
- примерное время чтения.

## Примеры запросов

Поиск фрагментов:

- `Найди, где говорится про первый вечер у Анны Павловны Шерер`
- `Найди фрагменты про Пьера Безухова`
- `Найди, где обсуждают Наполеона`

Вопросы:

- `Что произошло на вечере у Анны Павловны Шерер?`
- `Как в тексте представлен Пьер Безухов в начале романа?`
- `Что говорится о войне в первых главах?`

## Пример ожидаемого поведения

Если в библиотеке есть `warandpeace.txt`, бот должен:

- находить фрагменты из `Войны и мира` по тематическим запросам;
- отвечать на вопросы по книге;
- показывать цитаты, на которые опирается;
- честно писать, что ответа нет, если в книгах не найден релевантный контекст.

В репозитории уже лежит пример книги: `library/warandpeace.txt`.

## Замечания

- Для лучшего качества LLM-ответов используйте OpenRouter.
- Для нового набора книг ничего менять в коде не нужно: просто загружайте новые `.txt`.
- Если книга очень большая, первичная индексация может занять некоторое время.
